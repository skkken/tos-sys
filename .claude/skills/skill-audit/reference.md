# スキル監査 評価基準リファレンス

## 1. 技術的正確性（Technical Correctness）

スキルが使用するツール・APIが、Claude Codeの実際の挙動と整合しているか。

### 5点: 優秀
- 全てのツール使用が正確で、実際の挙動と完全に一致
- エージェント間通信モデルが適切（orchestrator-relay等）
- allowed-toolsと実際の使用ツールが一致

### 4点: 良好
- ツール使用はほぼ正確だが、軽微な不正確さが1-2箇所ある
- 動作に影響しない程度の記述の曖昧さがある

### 3点: 合格ライン
- 主要なツール使用は正しいが、一部に誤った前提がある
- 修正なしでも大部分は動作するが、一部のフェーズで問題が発生する可能性

### 2点: 不合格
- 重要なツールの使い方が間違っている（例: 終了済みエージェントへのSendMessage）
- スキルの主要フローが正しく動作しない可能性が高い

### 1点: 致命的
- 通信モデルの根本的な誤解がある
- 存在しない機能への依存がある
- スキルがほぼ動作しない

### チェックリスト
- [ ] allowed-toolsに列挙されたツールが全て実際に使用されているか
- [ ] 使用されている全ツールがallowed-toolsに含まれているか
- [ ] Agentツールの戻り値が正しく利用される設計か
- [ ] SendMessageを使用する場合、受信側エージェントが稼働中であることが保証されているか
- [ ] TaskCreate/TaskUpdateの依存関係が実際にフロー制御に寄与しているか（不要なら使わない）
- [ ] WebSearch/WebFetchで調査する内容が実際にWeb検索で確認可能か
- [ ] ファイル参照（${CLAUDE_SKILL_DIR}/...）が実在するファイルを指しているか
- [ ] エージェントのプロンプトにインライン展開の仕組みが明確に指示されているか

---

## 2. プロセス設計（Process Design）

ワークフローが目的達成の効率的な経路になっているか。

### 5点: 優秀
- 並行実行可能な箇所が全て並行化されている
- ユーザーチェックポイントが適切に配置されている
- エラー時・方向転換時のフローが明確
- 冗長なステップが一切ない

### 4点: 良好
- 全体的に効率的だが、並行化の余地が1箇所残っている
- ユーザーチェックポイントはあるが、配置が最適でない可能性

### 3点: 合格ライン
- フローは動作するが、明らかな非効率が1-2箇所ある
- ユーザーチェックポイントが不足している箇所がある

### 2点: 不合格
- 並行実行可能な箇所が全て直列化されている
- ユーザーチェックポイントがない
- 冗長なステップが複数ある

### 1点: 致命的
- ワークフローが目的達成に至らない
- 無限ループや到達不能なステップがある

### チェックリスト
- [ ] 依存関係のないステップが並行実行される設計か
- [ ] ユーザーの判断が必要なタイミングでチェックポイントがあるか
- [ ] 中間結果が多すぎる場合に絞り込むステップがあるか（全候補を評価しない等）
- [ ] ユーザーが方向転換したい場合に前のステップに戻れるか
- [ ] 各ステップの入出力が明確に定義されているか

---

## 3. 役割設計（Role Design）

エージェントのチーム構成と役割分担が適切か。

### 5点: 優秀
- 各エージェントの責務が明確に分離されている
- 役割間の重複がない
- エージェント数が最小限かつ十分
- 各エージェントのタスクスコープが現実的

### 4点: 良好
- 責務はほぼ分離されているが、軽微な重複が1箇所ある
- エージェント数は適切

### 3点: 合格ライン
- 一部に役割の重複があるが、全体としては機能する
- エージェント数が1名多い/少ない可能性

### 2点: 不合格
- 複数のエージェント間で大幅な役割重複がある
- エージェント数が明らかに過多または過少
- 1エージェントに非現実的な量のタスクが集中

### 1点: 致命的
- 役割分担が不明確で、誰が何をするか判別できない
- 必要な役割が欠けている

### チェックリスト
- [ ] 各エージェントの責務を1文で説明できるか
- [ ] 2つのエージェントの出力が大幅に重複しないか
- [ ] 統合可能な役割が不必要に分割されていないか
- [ ] 各エージェントのタスクが1回のAgent実行で現実的に完了可能か
- [ ] オーケストレーターの責務が明確か（中継・統合・判断のどれを担うか）

---

## 4. コンテンツ正確性（Content Accuracy）

スキル内の事実・参照・具体例が正確か。

### 5点: 優秀
- 全ての具体例・事実が正確
- ファイル間の参照が全て整合
- テンプレートがワークフロー出力と完全に一致

### 4点: 良好
- ほぼ正確だが、重要でない事実に1-2箇所の不正確さがある
- 参照・テンプレートは整合

### 3点: 合格ライン
- 主要な事実は正確だが、具体例に数箇所の誤りがある
- 参照やテンプレートに軽微な不整合がある

### 2点: 不合格
- 重要な事実の誤りがある
- ファイル間の参照に矛盾がある
- テンプレートとワークフロー出力が一致しない

### 1点: 致命的
- 主要な前提が事実と異なる
- 参照先ファイルが存在しない

### チェックリスト
- [ ] 具体例（サービス名の由来等）が事実と一致するか
- [ ] SKILL.mdから参照されるファイルが全て存在するか
- [ ] reference.mdの役割定義がSKILL.mdのチーム構成と一致するか
- [ ] テンプレートの項目がワークフローで生成される情報と対応しているか
- [ ] 評価基準・数値（候補数等）がファイル間で一貫しているか

---

## 5. プロンプト品質（Prompt Quality）

エージェントへの指示が明確で自己完結しているか。

### 5点: 優秀
- 全プロンプトが自己完結（外部参照なしで実行可能）
- 役割・作業内容・出力形式が明確に定義
- プレースホルダーの展開方法が具体的に説明されている

### 4点: 良好
- ほぼ自己完結だが、1箇所の記述が曖昧
- 出力形式は定義されている

### 3点: 合格ライン
- 作業内容は理解できるが、プレースホルダーの展開方法が一部不明確
- 出力形式の定義が不十分な箇所がある

### 2点: 不合格
- プロンプトが外部ファイルに依存し自己完結していない
- プレースホルダーが多数あり展開方法が不明
- 出力形式が定義されていない

### 1点: 致命的
- プロンプトの意図が不明確
- エージェントが何をすべきか判断できない

### チェックリスト
- [ ] 各プロンプトにサービス情報等の入力データが含まれる（または含める指示がある）か
- [ ] プレースホルダー（{...}）の全てに展開方法の説明があるか
- [ ] 出力形式が具体的に指定されているか
- [ ] エージェントの役割と制約が冒頭で明示されているか
- [ ] 前段のエージェント結果を含める手順が明確か
