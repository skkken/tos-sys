# Implementation Plan

- [x] 1. データベース・ストレージ基盤セットアップ
- [x] 1.1 領収書テーブルのマイグレーション作成
  - receiptsテーブルを作成する（id, amount, date, store_name, payment_method, storage_path, created_at, updated_at）
  - 金額は正の整数制約、payment_methodは'card','cash','other'のチェック制約を設定
  - 日付降順のインデックスを作成
  - RLSを有効にし、認証済みユーザーが全操作可能なポリシーを設定
  - _Requirements: 4.1, 4.4_

- [x] 1.2 Supabase Storageバケットのセットアップ
  - receiptsバケットをprivateで作成
  - 認証済みユーザーによるアップロード・閲覧・削除のRLSポリシーを設定
  - _Requirements: 1.3_

- [x] 1.3 Anthropic SDKのインストールと環境変数設定
  - @anthropic-ai/sdkパッケージを追加
  - ANTHROPIC_API_KEY環境変数の設定ガイドを.env.exampleに追記
  - OCR_MODEL環境変数（デフォルト: claude-sonnet-4-5-20250514）を設定可能にする
  - _Requirements: 2.1_

- [x] 2. サービスレイヤー実装
- [x] 2.1 (P) Storage操作サービスの実装
  - 画像アップロード機能を実装（ファイルパス: receipts/{userId}/{timestamp}_{fileName}）
  - 画像の署名付きURL取得機能を実装（時限アクセス用）
  - 画像削除機能を実装
  - サーバーサイドクライアント（service_role_key）を使用
  - _Requirements: 1.3, 5.5_

- [x] 2.2 (P) OCR解析サービスの実装
  - 画像をbase64エンコードしてClaude Vision APIに送信する機能を実装
  - 日本語領収書から金額（整数・円）、日付（YYYY-MM-DD）、店名を抽出するプロンプトを設計
  - 解析結果をJSON形式（amount, date, storeName）で返却
  - 解析失敗時はnull値を返却し、例外を投げない設計にする
  - max_tokensを256に制限（短い構造化レスポンスのみ）
  - _Requirements: 2.1, 2.5_

- [x] 2.3 (P) Zodバリデーションスキーマの定義
  - 領収書作成リクエストのバリデーションスキーマを定義（amount: 正の整数, date: YYYY-MM-DD, store_name, payment_method, storage_path）
  - 領収書更新リクエストのバリデーションスキーマを定義（各フィールドoptional）
  - 既存のparseRequest/parseBodyユーティリティと統合
  - _Requirements: 4.4, 3.2_

- [x] 3. APIルート実装
- [x] 3.1 画像アップロード＋OCR解析APIの実装
  - FormDataで画像ファイルを受け取るRoute Handlerを作成
  - withAuthで認証チェックを適用
  - ファイルサイズ（5MB以下）とMIMEタイプ（image/jpeg, image/png）をサーバー側で検証
  - Storageサービスで画像を保存し、OCRサービスで解析を実行
  - OCR結果と保存パスをレスポンスとして返却
  - OCR失敗時もアップロード成功として扱い、null値で返す
  - _Requirements: 1.3, 1.4, 1.5, 1.6, 2.1, 2.4, 2.5_

- [x] 3.2 (P) 領収書CRUD API（一覧・作成）の実装
  - 一覧取得API: 日付降順で取得、ページネーション対応
  - 作成API: Zodバリデーション→DB保存→201レスポンス
  - withAuthで認証チェックを適用
  - エラー時は既存のapiErrorパターンで処理
  - _Requirements: 4.1, 4.2, 4.3, 5.1, 5.2_

- [x] 3.3 (P) 領収書CRUD API（詳細・更新・削除）の実装
  - 詳細取得API: IDで取得、画像の署名付きURLも返却
  - 更新API: Zodバリデーション→DB更新
  - 削除API: 確認後にDB削除＋Storage画像削除
  - 存在しないIDには404を返却
  - _Requirements: 5.3, 5.4, 5.5_

- [x] 4. UIコンポーネント実装
- [x] 4.1 画像撮影・選択コンポーネントの実装
  - カメラ撮影ボタン（モバイルでリアカメラ起動）とギャラリー選択ボタンを作成
  - 選択された画像のプレビュー表示機能を実装
  - クライアント側で画像を5MB以下にリサイズする処理を実装（Canvas API）
  - FormDataでアップロードAPIに送信し、結果を親コンポーネントに通知
  - アップロード中のローディングインジケーターを表示
  - アップロード失敗時のエラーメッセージとリトライ機能を実装
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 2.3_

- [x] 4.2 領収書入力フォームコンポーネントの実装
  - OCR解析結果をフォームに自動入力する機能を実装
  - 金額（数値入力）、日付（日付入力）、店名（テキスト入力）の編集可能フォーム
  - 立替区分（カード・現金・その他）の選択UIをshadcn/uiで実装
  - 立替区分の選択を必須バリデーションとして設定
  - OCR解析失敗時に手動入力を促すメッセージを表示
  - 保存ボタンで領収書作成APIを呼び出し、成功時に一覧画面へ遷移
  - 保存失敗時のエラー表示とリトライ機能を実装
  - _Requirements: 2.2, 2.4, 3.1, 3.2, 3.3, 4.1, 4.2, 4.3_

- [x] 5. ページ実装・統合
- [x] 5.1 領収書新規登録ページの実装
  - 画像撮影コンポーネントとフォームコンポーネントを統合した新規登録ページを作成
  - 撮影→OCR解析→フォーム入力→保存の一連のフローを接続
  - モバイルファーストのレスポンシブレイアウトを適用
  - _Requirements: 1.1, 1.2, 2.2, 6.1, 6.2, 6.3_

- [x] 5.2 (P) 領収書一覧ページの実装
  - サーバーコンポーネントで領収書データを日付降順で取得し一覧表示
  - 各行に金額、日付、店名、立替区分を表示
  - 行タップで詳細ページへ遷移するリンクを設定
  - レスポンシブレイアウト（モバイル: カード、デスクトップ: テーブル）を適用
  - _Requirements: 5.1, 5.2, 6.1, 6.2_

- [x] 5.3 (P) 領収書詳細・編集・削除ページの実装
  - サーバーコンポーネントで領収書データと署名付き画像URLを取得し表示
  - 編集モードへの切り替えで金額・日付・店名・立替区分を編集可能にする
  - 削除ボタンで確認ダイアログを表示し、確認後にデータと画像を削除
  - レスポンシブレイアウトを適用（モバイルでタッチ操作に適したボタンサイズ確保）
  - _Requirements: 5.3, 5.4, 5.5, 6.1, 6.2_
